# 系统概述

BettaFish（微舆）是一个创新型多智能体舆情分析系统，通过多个专业 AI Agent 的协作，实现对社交媒体、新闻网站和私有数据库的深度分析与报告生成。

## 什么是 BettaFish？

BettaFish 是一个从零实现的智能分析平台，它能够：

- **自动采集**：从微博、小红书、抖音、快手等 10+ 社交平台采集数据
- **智能分析**：使用多个专业 Agent 进行多角度分析
- **协作推理**：通过"论坛"机制让 Agent 之间相互辩论和补充
- **报告生成**：自动生成结构化的 HTML/PDF/Markdown 报告

### 为什么叫 BettaFish？

BettaFish（暹罗斗鱼）是一种体型很小但非常好斗、漂亮的鱼，象征着"小而强大，不畏挑战"。这正是我们系统的设计理念——用轻量化的架构实现强大的分析能力。

> 💡 **设计哲学**：小而强大（Small but Mighty）——每个组件都保持精简，但组合起来可以实现复杂的多智能体协作。

## 设计理念

### 1. Agent 专业化 ⭐

每个 Agent 都有专门的工具集和能力边界，避免全能模型的同质化问题。

```
QueryAgent  → TavilyNewsAgency    → 新闻/网页搜索
MediaAgent  → BochaMultimodal     → 多模态内容理解
InsightAgent → MediaCrawlerDB     → 数据库深度挖掘
```

**设计优势**：
- ✅ 每个Agent专注于自己擅长的领域
- ✅ 工具集边界清晰，易于调试和扩展
- ✅ 避免全知模型的幻觉和混淆

### 2. 论坛协作机制 ⭐⭐

创新的"论坛"机制让不同 Agent 能够：

- **独立思考**：每个 Agent 基于自身工具进行分析
- **相互交流**：通过 forum.log 共享分析结果
- **主持人引导**：LLM 主持人总结争议点并引导深入讨论
- **迭代优化**：多轮讨论让分析更加深入

**论坛日志示例**：
```
[10:00:01] [QUERY] 根据新闻搜索，武汉大学近期在教育领域...
[10:00:02] [MEDIA] 从社交媒体图片看，校园环境优美...
[10:00:03] [INSIGHT] 数据库分析显示，用户对教学质量评价正面...
[10:00:04] [HOST] 三位专家都关注了武汉大学的声誉，建议下一轮重点讨论...
```

### 3. 渐进式报告生成 ⭐⭐

ReportEngine 采用多阶段生成流程：

```
模板选择 → 布局设计 → 篇幅规划 → 章节生成 → IR装订 → 渲染输出
```

每个阶段都由专门的 Node 处理，确保报告质量。

### 4. 公私域融合 ⭐⭐

系统不仅分析公开舆情，还支持：

- 私有数据库接入
- 业务数据融合分析
- 垂直场景定制

## 适用场景

### 1. 品牌声誉监控 ⭐

实时监测品牌在各平台的舆论，及时发现负面信息。

**典型问题**：
- "我们品牌在社交媒体上的形象如何？"
- "最近有什么负面舆情需要关注？"
- "用户对我们新产品的反响如何？"

### 2. 竞品分析 ⭐⭐

分析竞争对手的产品、营销策略和用户反馈。

**典型问题**：
- "竞品最近的营销活动效果如何？"
- "用户对我们和竞品的评价有什么差异？"
- "竞品在哪些方面做得比我们好？"

### 3. 政策影响评估 ⭐⭐⭐

分析政策发布后的社会反响和影响范围。

**典型问题**：
- "某政策发布后的舆论反响如何？"
- "政策对不同群体的影响有何差异？"
- "政策实施后的舆情趋势是什么？"

### 4. 危机公关 ⭐⭐⭐⭐

突发事件下的快速舆情分析和应对建议生成。

**典型问题**：
- "突发事件的舆情态势如何？"
- "应该如何回应公众关切？"
- "舆情发展可能如何演变？"

### 5. 市场研究 ⭐⭐

了解用户需求、痛点和市场趋势。

**典型问题**：
- "目标用户群体的主要关注点是什么？"
- "市场上有哪些未被满足的需求？"
- "用户对我们产品的使用体验如何？"

### 6. 私有数据分析 ⭐⭐⭐

将内部数据与外部舆情结合，获得更全面的洞察。

**典型问题**：
- "内部销售数据与外部舆情的关系？"
- "如何结合业务数据和社交媒体反馈？"

## 系统优势

| 优势 | 说明 | 对比传统方案 |
|------|------|-------------|
| **全域覆盖** | 支持 10+ 主流社媒平台 | 传统方案通常只覆盖 1-2 个平台 |
| **多模态能力** | 不仅能分析文本，还能理解视频和图片内容 | 传统方案大多只处理文本 |
| **智能协作** | Agent 论坛机制避免单一视角的盲点 | 传统方案使用单一模型，易产生盲点 |
| **可定制性** | 模块化设计，易于接入自定义数据和模型 | 传统方案通常难以扩展 |
| **轻量化** | 纯 Python 实现，一键部署，资源占用低 | 传统方案往往依赖重量级基础设施 |

- C → QueryEngine（新闻搜索）

### 练习 2：理解论坛机制 ⭐⭐

**问题**：以下哪些是论坛机制的核心特点？

A. 每个 Agent 使用相同的工具集
B. 主持人由人工控制
C. Agent 通过共享日志进行交流
D. 每 5 条发言触发一次主持人发言

<details>
<summary>查看答案</summary>

**正确答案：C、D**

解析：
- A 错误：每个 Agent 有专门的工具集
- B 错误：主持人由 LLM 自动生成
- C 正确：Agent 通过 forum.log 共享结果
- D 正确：每 5 条 Agent 发言触发一次主持人
</details>
